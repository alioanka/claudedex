What is the Arbitrage Module?

The Arbitrage Module is a multi-chain, multi-strategy system that detects and executes profitable price discrepancies across decentralized exchanges. It runs three parallel engines:

Ethereum Engine

Direct arbitrage between DEXs on Ethereum

  • Uniswap V2/V3, SushiSwap, Curve
  • 20+ token pairs monitored
  • Flash loans via Aave V3
  • Flashbots MEV protection
Solana Engine

High-speed arbitrage on Solana

  • Jupiter aggregator integration
  • 16+ token pairs including LSTs
  • Lower fees = lower thresholds
  • Jito MEV protection
Triangular Engine

Multi-hop cycle arbitrage

  • A → B → C → A profit cycles
  • 20+ triangular routes
  • Stablecoin & LST triangles
  • Gas-aware execution
How It Works
1 Price Monitoring

Every 2 seconds, the engines query prices from multiple DEX routers for each token pair.

2 Spread Detection

Calculate price difference between best buy and best sell DEX. Factor in gas costs dynamically.

3 Profitability Check

If spread > threshold (after gas + fees), opportunity is flagged for execution.

4 Atomic Execution

Execute via flash loan: Borrow → Buy low → Sell high → Repay → Keep profit. All in one tx.

5 MEV Protection

Submit via Flashbots (ETH) or Jito (Solana) to prevent front-running.

Flash Loan & Borrowing Configuration Guide
What are Flash Loans?

Flash loans allow you to borrow funds without collateral, as long as you repay within the same transaction. This enables arbitrage with zero capital at risk - you borrow, execute the arbitrage, and repay, all atomically.

Ethereum Flash Loans (Aave V3)
Status: Fully Implemented
  • Provider: Aave V3 Pool
  • Contract: 0x87870Bca3F3fD6335C3F4ce8392D69350B4fA4E2
  • Fee: 0.09% of borrowed amount
  • Max Borrow: Limited by pool liquidity (typically 100+ ETH)
How to Enable
  1. Go to Advanced tab above
  2. Enable "Use Flash Loans (Aave V3)"
  3. Set Flash Loan Amount (default: 10 ETH)
  4. Enable "Use Flashbots" for MEV protection
  5. Set DRY_RUN=false for live trading
Execution Flow:
1. Borrow X ETH from Aave V3 (0.09% fee)
2. Swap ETH → Token on DEX A (lower price)
3. Swap Token → ETH on DEX B (higher price)
4. Repay X ETH + 0.09% fee to Aave
5. Keep profit (spread - fees - gas)
Solana Borrowing Options
Important: Solana does NOT have native flash loans like Ethereum's Aave. The architecture is different.
Current Approach: Jupiter Aggregator

The Solana engine uses Jupiter's aggregated routing which finds the best swap path across all DEXs automatically.

  • No borrowing needed: Uses your SOL directly
  • Lower fees: Solana txs cost ~$0.001
  • MEV Protection: Jito bundles prevent front-running
Alternative: Solana Lending Protocols

If you want to borrow SOL to increase position size:

  • Solend: Borrow SOL with collateral (USDC, mSOL)
  • Marginfi: Leverage lending protocol
  • Kamino: Concentrated liquidity + lending
⚠️ These require collateral (not flash loans). Integration is planned for future updates.
How to Configure Solana Arbitrage
  1. Go to General tab, set Solana Trade Amount (e.g., 1-10 SOL)
  2. Go to Chains tab, enable Solana Arbitrage
  3. Enable Jupiter and Raydium DEXs
  4. Enable Jito MEV Protection for live trading
  5. Set SOLANA_PRIVATE_KEY in .env for live execution

Flash Loan Risks
  • Atomic Failure: If any step fails, the entire transaction reverts. You lose gas fees but no borrowed funds.
  • MEV Competition: Other bots watch for profitable opportunities. Use Flashbots to avoid front-running.
  • Smart Contract Risk: Bugs in the arbitrage contract could lead to stuck funds.
  • Pool Liquidity: Large flash loans may exceed available liquidity.
  • Gas Spikes: During network congestion, gas costs may exceed profits.

Quick Start Guide
1 Configure RPC URLs

Add at least ETHEREUM_RPC_URL to your .env file. For Solana, add SOLANA_RPC_URL.

2 Set DRY_RUN=true

Always start in simulation mode to verify the module works correctly.

3 Monitor the Logs

Check logs/arbitrage/arbitrage.log for opportunities and trades.

4 Review Dashboard

Use /arbitrage/dashboard to see stats and recent trades.

5 Enable Engines Selectively

Start with Ethereum only, then add Solana and Triangular once comfortable.

6 Configure MEV Protection

Before going live, set up Flashbots (ETH) or Jito (Solana) to prevent front-running.

7 Go Live (Carefully)

Set DRY_RUN=false, add wallet keys, start with small amounts.

Risk Warning
  • MEV Competition: Professional bots compete for the same opportunities. Without MEV protection, you may be front-run.
  • Smart Contract Risk: Flash loan contracts can have vulnerabilities.
  • Gas Wars: In competitive markets, gas costs can exceed profits.
  • Slippage: Large trades move prices, reducing actual profit.
  • Capital at Risk: Live trading can result in losses. Only trade what you can afford to lose.
Best Practices
  • Start Small: Begin with 0.1-1 ETH/SOL trade sizes until you understand the system.
  • Monitor Gas: High gas = lower profits. Consider pausing during gas spikes.
  • Diversify: Enable multiple engines (ETH + Solana + Triangular) for more opportunities.
  • Use Private RPC: Public RPCs have rate limits. Use Alchemy, Infura, or QuickNode.
  • Check Logs Daily: Review trades and opportunities to optimize thresholds.
  • Enable MEV Protection: Always use Flashbots/Jito for live trading.
